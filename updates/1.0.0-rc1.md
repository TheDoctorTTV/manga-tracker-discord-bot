# 1.0.0-rc1

Release candidate 1 for the `beta` branch, based on the changes currently ahead of `main` as of 2026-03-12.

## Highlights

- Added a full web dashboard experience with onboarding, login, status, and tabbed admin flows.
- Introduced Discord OAuth and guild-scoped dashboard authentication for safer administrative access.
- Added dashboard-managed environment configuration and a built-in GitHub release updater.
- Refactored manga tracking into modular services and source adapters, including Comix and MangaDex support.
- Reworked Linux packaging and systemd installation around binary and release package workflows.

## Added

- Dashboard pages and supporting server logic for onboarding, login, status, and admin control flows.
- `dashboardAuthService`, `discordOAuthService`, `envFileService`, and `githubReleaseUpdater` services.
- `mangaTrackerService` plus source adapter infrastructure under `src/sources/`.
- Linux release assets, including install and uninstall scripts and release package documentation.
- Test coverage for auth, onboarding, tab state, environment file management, updater behavior, and guild tracking flows.

## Changed

- Replaced the older single-file bot flow with a modular app entrypoint and bot service structure.
- Updated README and example environment configuration to match the onboarding-first setup flow.
- Split packaging into release and prerelease build scripts and added version management tooling.
- Refined dashboard styling, accessibility, tab restore behavior, and packaged asset loading.
- Improved self-update behavior, including asset refresh, restart handling, and manual recovery guidance.

## Removed

- Removed legacy bootstrap, setup, and update shell flows that were superseded by the packaged installer workflow.
- Removed the older monolithic `manga-chapter-updates.js` implementation in favor of the modular service architecture.
